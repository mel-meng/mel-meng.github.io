    Assumptions used for this export A compact reference for QA, demos, and repeatability.

    Units and ICM import

    XPX source (this file)

    Per-node DRY_WTH_UN + T_SEWAGE — not inferred from flow magnitude.

      ICM subcatchment CSV

      additional_foul_flow → cfs

      ICM Waste Water event CSV

      Profile multipliers; template average-flow unit → ft³/day

      Profile header FLOW is exported as 0.

      Export assumptions

      Subcatchment CSV columns

        x, y from XPX NODE line (same as node DWF info CSV).

        Row inclusion

        One row per node with DWF loading (hasDwfLoading) — same rule as explorer filter.

        Not exported to ICM

        • T_DWF_PFAC (peaking factor)
        • Census / area-only DWF (T_ASUB, T_POPDEN without T_SEWAGE)
        • Unreferenced temporal variations
        • Network node/link design-input export (separate workflow — Additional CSV Export)

        Waste Water event CSV

        • Profile header FLOW = 0; shape in calibration multipliers
        • wastewater_profile is integer PROFILE_NUMBER, not pattern name
        • Profiles: referenced pattern names only; stable alphabetical numbering
        • Hourly multipliers rotated for ICM 0-based hours (00:00 = XPX H24, then H1…H23)
